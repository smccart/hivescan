# Configuration

HiveAgents is zero-config by default: it auto-discovers all directories under `apps/` and `packages/` in your project root. For most monorepos, that's enough to get started.

When you want control over which agents appear, their display names, port links, and colors, create a `hive.config.js` in your monorepo root.

## hive.config.js

```js
// hive.config.js
export default {
  port: 4199,           // optional — default 4199
  defaultModel: 'claude-opus-4-6', // optional — model to use on startup
  agents: [
    {
      name: 'web',      // unique key (used internally)
      dir: 'apps/web',  // path relative to repo root
      label: 'Web App', // display name in the sidebar
      port: 3000,       // optional — shows as a clickable link in the header
      color: '#4ade80', // optional — hex color for status indicators
    },
    {
      name: 'api',
      dir: 'apps/api',
      label: 'API Server',
      port: 8000,
      color: '#60a5fa',
    },
    {
      name: 'admin',
      dir: 'apps/admin',
      label: 'Admin',
      port: 3001,
    },
  ],
}
```

## Config options

### Top-level

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `port` | `number` | `4199` | Port for the HiveAgents web server |
| `defaultModel` | `string` | `'claude-opus-4-6'` | Claude model to use when spawning agents |
| `agents` | `Agent[]` | auto-discovered | List of agent configurations |

### Agent options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `name` | `string` | Yes | Unique identifier. Used as the buffer file name and WebSocket param |
| `dir` | `string` | Yes | Directory path (relative to repo root) where `claude` will run |
| `label` | `string` | No | Display name shown in the sidebar. Defaults to `name` |
| `port` | `number` | No | If set, shown as a clickable link in the terminal header |
| `color` | `string` | No | Hex color for status dots and active indicator. Auto-assigned from palette if omitted |

## Auto-discovery

If no `hive.config.js` is found, HiveAgents scans:
- `./apps/*/` — all subdirectories of `apps/`
- `./packages/*/` — all subdirectories of `packages/`

Each discovered directory becomes an agent. Colors are assigned from a built-in palette. Labels are generated by converting directory names from kebab-case/snake-case to Title Case.

## Buffer storage

Session buffers are stored in `.hive/` at your repo root. Add it to `.gitignore`:

```
.hive/
```

Each agent's buffer is capped at 50 KB (rolling — oldest output is trimmed first).

## Environment

HiveAgents resolves Claude's path and your login-shell `PATH` automatically, so tools like `nvm`, `fnm`, and `mise` work as expected inside the spawned agent processes.

## Claude Code permissions

HiveAgents can manage your project's `.claude/settings.json` file, which controls how Claude Code handles tool permissions. You can edit this from the dashboard UI (click **Permissions** in the sidebar) or by hand.

```json
{
  "defaultMode": "acceptEdits",
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git diff *)",
      "Bash(git commit *)",
      "Read"
    ],
    "deny": [
      "Bash(git push *)",
      "Bash(rm -rf *)",
      "Read(.env*)"
    ]
  }
}
```

### Default modes

| Mode | Behavior |
|------|----------|
| `default` | Prompts for each tool on first use |
| `acceptEdits` | Auto-approves file edits, prompts for shell commands |
| `plan` | Read-only — analyze but don't modify |
| `dontAsk` | Auto-denies unless listed in allow rules |
| `bypassPermissions` | Skips all prompts (isolated environments only) |

### Rule syntax

Rules follow the pattern `Tool(pattern)` where `Tool` is `Bash`, `Edit`, `Read`, or `Write`, and `pattern` is a glob:

- `Bash(npm run *)` — matches any `npm run` command
- `Edit(/src/**)` — matches edits to any file under `src/`
- `Read` — matches all file reads (no pattern = match all)
- `Read(.env*)` — matches reads of `.env`, `.env.local`, etc.

Deny rules take priority over allow rules.
